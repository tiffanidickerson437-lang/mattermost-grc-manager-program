# 04 · AI governance

One artifact lives here: [`internal-ai-policy.md`](internal-ai-policy.md), a draft internal AI usage policy for Mattermost, written in the human-in-the-loop vocabulary Mattermost's own engineers published in the [multiplayer tool-calling specification](https://mattermost.com/blog/multiplayer-tool-calling-for-secure-operations-who-approves-who-sees-who-runs-the-tool/) behind Agents V2.

Why this exists: based on my research, the handbook's `llms.txt` index (281 pages, checked 17 July 2026) contains no AI usage policy, no LLM data-handling standard, and no AI governance page, while the company publishes [AI risk guidance for other organizations' CISOs](https://mattermost.com/blog/sovereign-ai-risk-assessment-ciso-questions/). Whether a policy exists internally is not knowable from outside; that is [question 6](../00-governance/open-questions.md). If one exists, this draft is a review input. If none exists, it is a fast early win.

The core mapping the draft is built on:

> observation automates · assertion asks · attestation never automates
