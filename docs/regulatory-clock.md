# The regulatory clock

Every date that governs the plan. All verified against primary sources on 16 July 2026. Forward dates marked as estimates where the source says so.

## What already happened

| Date | Event | Why it matters here |
|---|---|---|
| 2025-11-10 | CMMC Phase 1 live: DFARS 252.204-7021 becomes a contract gate | Self-assessment requirements in force. Never suspended, still binding |
| 2026-02-25 | FedRAMP NTC-0004: rename announced (Authorization becomes Certification; impact levels become Classes A/B/C/D) | The vocabulary shift most of the market has not absorbed |
| 2026-03-03 | FedRAMP NTC-0007: SOC 2 Type II accepted as entry to Class A, via Program Certification, no agency sponsor | A live strategic option for any SOC 2 Type II holder. Two hard limits: no reciprocity toward higher classes, and FedRAMP states open concerns about SOC 2 audit quality |
| 2026-03-06 | NTC-0008: Rev5 Program Certifications, no sponsor required | The sponsorship bottleneck removed for the traditional path too |
| 2026-05-04 | FedRAMP rename executed | "FedRAMP High / Authorized" is retired vocabulary from this date. Classes A/B/C/D and "Certified" are current |
| 2026-06-24 | CR26 released: consolidated rules, valid to 2028-12-31, with JSON schemas for machine-readable packages | The federal cloud program now validates submissions with software. Patched 7/14; in-browser validator shipped 7/15 |
| 2026-07-02 | Mattermost publishes the GRC Manager req | Metric 1 assumes the pre-suspension CMMC world |
| 2026-07-13 | DoW CIO suspends CMMC Phase II (memo signed 7/10, Kirsten A. Davies) | Only Level 1 (Self) and Level 2 (Self) may be required in contracts. C3PAO and DIBCAC designations prohibited. Active solicitations amended to remove them. No waivers during the review. DFARS 7012 and NIST 800-171 Rev 2 explicitly remain enforced |

## The two sentences worth keeping loaded

From the suspension's Task Force mandate, as reported by Federal News Network: recommendations must produce a framework that "prioritizes speed to capability, lowers barriers for small, medium, and non-traditional businesses, and replaces prohibitive, third-party compliance models with scalable, realistic security measures."

From [fedramp.gov/20x](https://www.fedramp.gov/20x/), Core Principles, Automatic Validation: "Continuous evidence of what is happening is stronger than a policy saying it should happen."

Both federal regimes, in their own words, are moving to the operating model this role's posting asks for, which means the case for continuous, automated, evidence-computed compliance no longer needs to be argued from first principles. The regulators have already made it.
